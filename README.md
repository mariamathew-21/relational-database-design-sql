# Relational Database Design and SQL Analysis

## Project Overview
This project demonstrates the design and implementation of a relational database using a publicly available dataset. The objective was to design a structured database system, apply normalization techniques, implement the schema using SQL, and perform analytical queries to extract meaningful insights.

## Key Tasks Completed

### Data Understanding
Investigated and analysed a publicly available dataset to determine the entities, attributes, and relationships required for database design.

### Database Design
Created an Entity-Relationship (ER) diagram representing the conceptual database structure, including entities, relationships, and key attributes.

### Normalization
Applied normalization techniques to reduce redundancy and ensure data integrity within the relational database schema.

### Database Implementation
Implemented the database schema using SQL scripts and imported the dataset into the database system.

### Data Querying and Analysis
Developed multiple SQL queries to retrieve and analyse data based on specific requirements.

### Data Visualization
Visualised query outputs using graphical representations to better interpret analytical results.
